Aim: - To study the DDL Commands.

 

1) CREATE :-


  •  Create Command is used to create a table in the database .

  • The CREATE TABLE command defines each column has minimum of three attributes .

  • Name
  • Data Type
  • Size (Column Width )

Syntax :- 

                            CREATE TABLE table_name (
    column_name1 data_type(size),
    column_name2 data_type(size);
);

Ex :- 

                            CREATE TABLE persons (
    Person_Id  int ,
    Last_Name varchar(225) ,
    First_Name varchar(225) ,
    Address varchar(225) ,
    City varchar(225) ,
);

2) ALTER


  • ALTER Command will be used to modify the structure of the existing table using ALTER we can do following things .
  • Add new columns to the table .

    Syntax :- 
                                    ALTER TABLE student ADD ( Location varchar(30));
  • Removing existing column from table .

    Syntax :- 
                                    ALTER TABLE <table_name> DROP COLUMN column_name 
    ALTER TABLE student DROP COLUMN course
  • Modify columns data type size of the table

    Example :-   
    ALTER TABLE Student MODIFY (Name varchar(20));
    • Change the name of the column of the table :-

    Syntax :- 
                                    ALTER TABLE <table_name> RENAME COLUMN <Old_name> TO <New_name>
    Example :- 
                                    ALTER TABLE student RENAME COLUMN Location TO Branch

3) TRUNCATE


  • TRUNCATE command used to remove the data from the table .
    Syntax :- 

    TRUNCATE TABLE <table_name>
    Ex :- 
                                      TRUNCATE TABLE Student

4) DROP


  • DROP Command used to remove table from the database .
  • It removes entire table from the database along with data present in it .

    Syntax :- 

    DROP TABLE <table_name>
    Ex :- 
                                      DROP TABLE Student

5) RENAME


  • RENAME Command used to change the name of database objects like table , view etc.
  • Using this command we can change only the name of database object not the columns of the table .

    Syntax :- 

     RENAME <Old_name> TO <New_name>
    Ex :- 
                                      RENAME Student TO Student_data

Aim: - To study the DML Commands.
    Resources used Oracle 10g or SQL Server 2005 .

Theory: -

  • DML Commands are used to modify the data presented in the table.
  • These statement work on the data.
  • DML requires transaction control, any changes which is made to the database by DML statement will be called as transaction.
  • Any changes made by DML statement controlled by TCL statement .
  • DML Support following Categories of Commands ,
    1. INSERT
    2. DELETE
    3. UPDATE

1) INSERT: -

  • INSERT command will be used to insert new data into table .
  • In other words it will be used to create new rows in table .
  • While using this command the values are separated by commas & the values of data type varcahar , char , long are enclosed in single quotes.
  • We have 2 different syntax for writing Insert.
Syntax: - 

 INSERT INTO <table_name> VALUES (a_list_of_values)

Example: -

INSERT INTO dept VALUES (50,'IT','Pune');

   If data has to be written into only specific column this too is possible with slight variations to the general syntax.

Syntax:- 
INSERT INTO <table_name(column_name)> VALUES (a_list_of_values);
Syntax:- 

INSERT INTO emp(empno,ename,job,sal,hiredate) 
                  VALUES (101,'Vivek','Professor','40000',sysdate);

2) UPDATE: -

  • UPDATE command will be used to modify the data present in tables .
  • The update command consist of a ' set ' clause and an optional ' where ' clause to reflect changes in existing data UPDATE command is used .

Syntax: -

UPDATE <table_name> SET field1=value,field2=value WHERE condition;
Example: -

UPDATE dept SET dname='ACCOUNT' WHERE deptno=50 ;


Note:- If the " WHERE " condition is committed , the entire table is committed.


Eg.

 UPDATE emp SET job='DBA', sal=40000 WHERE empno=101;
Eg.

 UPDATE emp SET sal=(SELECT sal FROM emp WHERE empno=101) WHERE empno=7902;



3) DELETE: -

  • DELETE command will be used to remove data from table.
  • Using DELETE we can remove single row or rows from table.
  • The DELETE command consist the optional. ' where ' clause.
  • Two way we can delete the data.
Syntax: -

DELETE from <table_name> WHERE <condition>

First Way: -
Eg. 

DELETE FROM emp;

Second Way: - 

DELETE FROM emp WHERE job='DBA';

Theory: -

SQL Integrity Constraints :


   SQL constraints are used to specify rules for the data in a table.

   Constraints are used to limit the type of data that can go into a table. This ensures the accuracy and reliability of the data in the table. If there is any violation between the constraint and the data action, the action is aborted.

   Constraints can be column level or table level. Column level constraints apply to a column, and table level constraints apply to the whole table.

   The following constraints are commonly used in SQL:

  • NOT NULL - Ensures that a column cannot have a NULL value
  • UNIQUE KEY - Ensures that all values in a column are different
  • PRIMARY KEY - A combination of a NOT NULL and UNIQUE. Uniquely identifies each row in a table
  • FOREIGN KEY - Prevents actions that would destroy links between tables
  • CHECK - Ensures that the values in a column satisfies a specific condition
  • DEFAULT - Sets a default value for a column if no value is specified

   Integrity Constraints are used to apply business rules for the database tables.

   # Constraints can be defined in two ways :

  1. The constraints can be specified immediately after the column definition. This is called column-level definition.
  2. The constraints can be specified after all the columns are defined. This is called table-level definition.

1) SQL Primary Key :

This constraint defines a column or combination of columns which uniquely identifies each row in the table.

Syntax to define a Primary key at column level:

column name datatype [CONSTRAINT constraint_name] PRIMARY KEY

Syntax to define a Primary key at table level:

[CONSTRAINT constraint_name] PRIMARY KEY (column name1,column name2...)

  • column_name1, column_name2 are the names of the columns which define the primary Key.
  • The syntax within the bracket i.e. [CONSTRAINT constraint_name] is optional.

For Example: To create an employee table with Primary Key constraint, the query would be like.

Primary Key at column level :

                  CREATE TABLE employee
 (id number(5) CONSTRAINT emp_id_pk PRIMARY KEY,
 name char(20),
 dept char(10),
 age number(2),
 salary number(10),
 location char(10)
 );

2) SQL Foreign Key or Referential Integrity :

This constraint identifies any column referencing the PRIMARY KEY in another table. It establishes a relationship between two columns in the same table or between different tables.

For a column to be defined as a Foreign Key, it should be a defined as a Primary Key in the table which it is referring. One or more columns can be defined as Foreign key.

Syntax to define a Foreign key at column level: 


[CONSTRAINT constraint_name] REFERENCES Referenced
 Table name(column_name)
Syntax to define a Foreign key at table level:


 [CONSTRAINT constraint_name] FOREIGN KEY(column_name)
 REFERENCES referenced_table_name(column_name);

4) SQL Unique Key:

This constraint ensures that a column or a group of columns in each row have a distinct value. A column(s) can have a null value but the values cannot be duplicated.

Syntax to define a Unique key at column level:

[CONSTRAINT constraint_name] UNIQUE
Syntax to define a Unique key at table level:
